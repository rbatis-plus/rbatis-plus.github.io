## Status meanings

- **Public**：Anyone can read the repository; this does not automatically mean a stable artifact exists.
- **Private preview**：The project exists but is limited to authorized members; external visitors may receive a 404.
- **Archived**：Kept for history and not a default dependency for new projects.

## Before adopting a repository

1. Read the target repository README and releases.
2. Confirm its license, runtime baseline, and maturity statement.
3. Run the repository’s own gates in your environment; this catalog is not a substitute for verification.
